# Данные и контент

Весь текстовый контент сайта вынесен из компонентов в типизированные файлы. Компоненты не содержат «зашитых» строк, поэтому правка текста и перевод не требуют касания вёрстки.

## Где что лежит

```
src/content/
  types.ts   # тип SiteContent — единая схема всего контента
  ru.ts      # русская версия (объект типа SiteContent)
  en.ts      # английская версия (объект типа SiteContent)
```

- `ru.ts` и `en.ts` обязаны соответствовать одному типу `SiteContent`. Если добавить поле в `types.ts`, TypeScript потребует заполнить его в обоих языках — переводы не разъезжаются.
- Доступ к контенту в компонентах — через хук `useContent()` (`src/hooks/useContent.ts`), который отдаёт объект текущего языка: `const { c, lang } = useContent()`.

## Схема `SiteContent`

| Поле | Тип | Содержимое |
|------|-----|-----------|
| `doctor` | объект | `name`, `role`, `location`, `portraitAlt` (alt-текст портрета). |
| `nav` | объект | Подписи пунктов меню: `services`, `prices`, `about`, `faq`. |
| `lang` | объект | `switchTo` (подпись/aria кнопки языка), `current`. |
| `hero` | объект | `badge`, `summary`, `ctaPrimary`, `ctaSecondary`, `chips[]`, `scrollCue`. |
| `stats` | массив | `{ value, label }` — карточки статистики. |
| `about` | объект | `eyebrow`, `heading`, `body[]` (абзацы), `credentials[]` (`{ period, title, place }`). |
| `services` | объект | `eyebrow`, `heading`, `note`, `hint`, `items[]` (`{ title, desc }`) — слайды. |
| `skills` | объект | `eyebrow`, `heading`, `note`, `groups[]` (`{ title, items[] }`) — навыки и качества тегами. |
| `visit` | объект | `eyebrow`, `heading`, `steps[]` (`{ title, desc }`). |
| `faq` | объект | `eyebrow`, `heading`, `items[]` (`{ q, a }`). |
| `contact` | объект | `eyebrow`, `heading`, `body`, `ctaPrimary`, `note`, `channels[]`. |
| `sticky` | объект | `label` — подпись плавающей кнопки. |
| `footer` | объект | `disclaimer`, `rights`. |

### Каналы связи (`contact.channels`)

```ts
{ label: string; value: string; href: string; kind: string }
```

`kind` определяет иконку и поведение ссылки:

| `kind` | Иконка | Ссылка |
|--------|--------|--------|
| `whatsapp` | сообщение | открывается в новой вкладке; используется как действие кнопки «Записаться» по умолчанию |
| `telegram` | самолётик | новая вкладка |
| `phone` | трубка | `tel:` в том же окне |
| `email` | конверт | `mailto:` в том же окне |
| `map` | пин | новая вкладка (карта) |

## Как редактировать

1. **Изменить текст** — найдите нужное поле в `ru.ts`/`en.ts` и отредактируйте строку. Меняйте обе версии.
2. **Добавить направление** — добавьте `{ title, desc }` в `services.items` (в обоих языках). Слайдер и точки навигации подстроятся автоматически.
3. **Добавить навык/качество** — добавьте строку в нужную группу `skills.groups[].items`.
4. **Добавить вопрос** — добавьте `{ q, a }` в `faq.items`.
5. **Сменить контакты** — отредактируйте `contact.channels` (и `value`, и `href`).

> Цен на странице нет: в исходном резюме нет данных о стоимости. Если понадобится прайс, верните секцию `prices` в `types.ts` и компонент по образцу `Skills.tsx`.

## Хранение языка

Выбранный язык сохраняется в `localStorage` под ключом `taplink-lang` (см. `src/i18n.ts`). При первом заходе язык определяется по `navigator.language` (англоязычные браузеры — `en`, остальные — `ru`). Атрибут `<html lang>` обновляется при переключении.

## Чего здесь нет

Бэкенда и базы данных нет: это статический фронтенд. «Данные» — это контент в TypeScript-файлах, который собирается в бандл. Заявки на приём уходят во внешний канал (мессенджер или подключённая система записи), а не в собственное хранилище.
