# Обзор приложения

Персональный таплинк (link-in-bio) для врача-педиатра (Кристина Кургинян, Москва). Одностраничный сайт с длинным скроллом, мобильный приоритет, два языка (RU/EN). Главная задача — превратить переход из профиля в соцсети в обращение и запись на приём.

## Что внутри

Одна страница, собранная из секций (сверху вниз):

| Секция | Файл | Назначение |
|--------|------|-----------|
| Шапка | `Header.tsx` | Лого-монограмма, навигация (десктоп), переключатель языка. Прозрачная, затемняется при скролле. |
| Hero | `Hero.tsx` | Имя, специальность, краткое описание, портрет (`Portrait.tsx`), две CTA-кнопки, чипы. |
| Факты | `Stats.tsx` | Честные факты: возраст пациентов, год начала практики, число учреждений, вуз. |
| О враче | `About.tsx` | Текст + таймлайн образования и опыта работы. |
| Помощь | `Services.tsx` | Горизонтальный слайдер направлений (scroll-snap, стрелки, точки). |
| Как проходит приём | `Visit.tsx` | Три пронумерованных шага. |
| Навыки | `Skills.tsx` | Профессиональные навыки и личные качества тегами в двух группах. |
| Вопросы | `Faq.tsx` | Аккордеон на нативном `<details>`. |
| Запись | `Contact.tsx` | Тёмная карточка с CTA и каналами связи (WhatsApp, телефон, email, город). |
| Подвал | `Footer.tsx` | Реквизиты и дисклеймер. |
| Плавающая кнопка | `StickyCta.tsx` | Кнопка «Записаться» снизу на мобильных. |

## Технологии

- **React 19 + TypeScript + Vite** — каркас и сборка.
- **framer-motion** — анимации появления секций, hero, слайдов.
- **lenis** — плавный инерционный скролл.
- **react-i18next + i18next** — переключение языков RU/EN.
- **lucide-react** — иконки.
- **CSS** — токены дизайна в `src/index.css`, стили секций в `src/styles/sections.css` (OKLCH, fluid `clamp()`).

## Дизайн

Направление — «тёплый кабинет частной клиники»: глубокий чернильный индиго несёт премиальность и доверие, тёплое латунное золото добавляет тепло, чистый светлый фон для чтения. Тепло, уместное для детского врача, идёт от цвета, типографики и фото, а не от детских мультяшных элементов. Подробности в [DESIGN.md](./DESIGN.md), стратегия и аудитория — в [PRODUCT.md](./PRODUCT.md).

Навигация: все переходы по якорям (меню, кнопки, «листайте вниз») плавные — перехватываются в `useSmoothScroll.ts` и прокручиваются через Lenis с отступом под шапку.

Доступность: контраст WCAG AA, видимый фокус, поддержка `prefers-reduced-motion` (анимации заменяются мягким фейдом, плавный скролл отключается и переходит в мгновенный с тем же отступом), тач-таргеты ≥44px.

## Запуск

```bash
cd app/front
npm install
npm run dev      # дев-сервер (http://localhost:5173)
npm run build    # продакшн-сборка в app/front/dist
npm run preview  # предпросмотр сборки
```

## Что нужно заменить перед публикацией

- **Портрет** — `public/portrait.png` сейчас сгенерированная заглушка (не реальное фото). Замените на настоящее фото Кристины. Если файла нет или он не загрузился, автоматически показывается абстрактная заглушка с монограммой (`Portrait.tsx`), белого пятна не будет.
- **Кнопка записи** — ведёт на WhatsApp. При желании подключите систему записи (Yclients, форму и т.д.) в `Contact.tsx` и `Hero.tsx`.
- **Цены** — секции с ценами нет: в исходном резюме нет данных о стоимости. Если нужен прайс, добавьте секцию (текст готов поддержать структуру в `content/`).
- **Тексты и контакты** — правятся в `src/content/ru.ts` и `src/content/en.ts` (см. [DATA.md](./DATA.md)). Телефон, WhatsApp и email подставлены из резюме.
- **Монограмма и favicon** — `public/favicon.svg`; инициалы в шапке выводятся из имени автоматически.

Реальные данные взяты из резюме (`RESUME.md`). Контакт рекомендателя на странице сознательно не публикуется (приватность).
